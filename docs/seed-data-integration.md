# Seed Data Integration Guide

This guide describes how frontend/admin pages should consume seeded backend data.

## Source of truth

- Seeded data is generated by backend (`D:\DoAn\panthora_be`) from JSON files in `src/Infrastructure/Data/Seed/Seeddata`.
- Admin pages in frontend must consume backend APIs and should not rely on local hardcoded datasets as the primary source.

## Admin data endpoint

- Backend endpoint: `GET /api/admin/overview`
- Frontend service: `frontend/src/services/adminService.ts`

This endpoint provides:

- dashboard statistics
- customers list
- payments list
- insurances list
- visa applications list

## Local verification checklist

1. Start backend and frontend containers:

```bash
wsl -d Ubuntu-22.04 -- sh -lc "cd /mnt/d/DoAn/panthora_be && docker compose up -d --build --no-deps api frontend"
```

2. Validate backend health and admin endpoint:

```bash
curl -I http://localhost:8080/health
curl -I http://localhost:8080/api/admin/overview
```

3. Validate frontend pages that consume backend admin data:

- `http://localhost:4001/dashboard`
- `http://localhost:4001/dashboard/customers`
- `http://localhost:4001/dashboard/payments`
- `http://localhost:4001/dashboard/insurance`
- `http://localhost:4001/dashboard/visa`

4. Validate frontend build:

```bash
cd frontend
npm run build
```

## Policy for new admin pages

- Prefer backend API data first.
- Use fallback demo/mock data only for temporary resilience while API is unavailable, and clearly mark it as fallback behavior.
- Any new backend seed class must have a corresponding JSON seed file and preflight validation entry.
